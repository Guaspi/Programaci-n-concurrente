**Práctica 1 – Ada (Programación Concurrente)**

En esta práctica se trabaja la concurrencia mediante tareas y mecanismos de sincronización propios de Ada, 
como tasks y analizando la coordinación entre procesos y el control de acceso a recursos compartidos.

**Práctica 2 – Go (Programación Concurrente)**

En esta práctica se implementan soluciones concurrentes utilizando goroutines y canales (channels), explorando el
modelo CSP y la comunicación segura entre hilos ligeros.

**Práctica 3 – Python (Programación Concurrente)**

Es la práctica más extensa y se centra en el uso de hilos, procesos y mecanismos de sincronización en Python
(threading, multiprocessing, locks, semáforos, etc.).  Se analizan problemas clásicos de concurrencia y la gestión
de recursos compartidos.
